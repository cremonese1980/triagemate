## 🅐 Design Freeze

Problem:
Durable idempotency (ADR-005) requires database infrastructure.

Solution:
Introduce PostgreSQL runtime for dev and CI,
wire datasource profiles,
enable health indicator.

Out of scope:
- No business tables
- No idempotency table yet
- No JPA entities

Decision:
PostgreSQL chosen (enterprise standard, Testcontainers support).

---

## 🅑 Implementation Tasks

### 9.1.a PostgreSQL Docker Runtime
- Add docker-compose service `postgres`
- Expose port 5432
- Add persistent volume (dev only)
- No admin UI by default

### 9.1.b Spring Datasource Profiles
- application-dev.yml:
  datasource url, username, password
- application-test.yml:
  overridden via Testcontainers
- No profile hacks like `!test`

### 9.1.c Actuator Health
- DB health must be visible in `/actuator/health`
- Startup fails fast if DB unavailable in dev

### 9.1.d Secrets Discipline
- No real credentials committed
- Defaults acceptable for local dev only
- Env override documented
