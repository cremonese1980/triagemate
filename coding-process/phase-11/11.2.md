## Objective

Establish robust correlation tracking across the entire event pipeline:

- **Ingestion boundary:** Populate MDC from Kafka message
- **Processing:** Maintain MDC through business logic
- **Outbox write:** Embed trace in JSONB payload
- **Outbox publish:** Restore MDC when publishing asynchronously
- **Thread safety:** Propagate MDC to async tasks and thread pools

This ensures every log statement can be correlated from event ingestion through decision execution to final event publication, even across thread boundaries.

### 11.2.a — Populate MDC at Kafka Boundary

#### Goal

Extract trace fields from incoming Kafka messages and populate MDC before any business logic executes.

This establishes the correlation context for all downstream processing.

---

#### Technical Implementation

**Step 1: Define EventEnvelope structure**

**Create:** `com.triagemate.triage.model.EventEnvelope`

```java
/**
 * Standard event envelope wrapping all events.
 *
 * Note: Uses JsonNode for payload to avoid coupling with specific event types.
 * Trade-off: Couples to Jackson in model layer, but enables generic deserialization.
 */
public record EventEnvelope(
                String eventId,
                String eventType,
                ZonedDateTime timestamp,
                Trace trace,
                JsonNode payload
        ) {
    /**
     * Trace context for correlation.
     *
     * Can be null for legacy events - callers must handle gracefully.
     */
    public record Trace(
            String requestId,
            String correlationId
    ) {}
}
```

**Why nullable Trace:**
- Legacy events may lack trace field
- Events from external systems may not have trace
- Null-safety required in all consumers

```java
@Component
public class EventEnvelopeDeserializer {

    private final ObjectMapper objectMapper;

    public EventEnvelopeDeserializer(ObjectMapper objectMapper) {
        this.objectMapper = objectMapper;
    }

    public EventEnvelope deserialize(String json) {
        try {
            return objectMapper.readValue(json, EventEnvelope.class);
        } catch (JsonProcessingException e) {
            throw new EventDeserializationException("Failed to deserialize event envelope", e);
        }
    }

    public <T> T extractPayload(EventEnvelope envelope, Class<T> payloadType) {
        try {
            return objectMapper.treeToValue(envelope.payload(), payloadType);
        } catch (JsonProcessingException e) {
            throw new EventDeserializationException("Failed to extract payload", e);
        }
    }
}
```

**Step 3: Implement MDC support utility**

**Create:** `com.triagemate.triage.kafka.MdcKafkaSupport`

```java
/**
 * Utility for MDC-aware Kafka message consumption.
 *
 * Enforces MDC lifecycle: populate → process → clear.
 * Handles null trace gracefully with generated fallbacks.
 */
public final class MdcKafkaSupport {

    private MdcKafkaSupport() {}

    /**
     * Process Kafka record with automatic MDC lifecycle management.
     *
     * @param record Kafka consumer record
     * @param payloadType Expected payload type
     * @param deserializer Envelope deserializer
     * @param processor Business logic processor
     * @param errorHandler Error handling callback (can log, send to DLQ, etc.)
     */
    public static <T> void consumeWithMdc(
            ConsumerRecord<String, String> record,
            Class<T> payloadType,
            EventEnvelopeDeserializer deserializer,
            Consumer<T> processor,
            BiConsumer<Exception, ConsumerRecord<String, String>> errorHandler) {

        try {
            // Deserialize envelope
            EventEnvelope envelope = deserializer.deserialize(record.value());

            // Populate MDC (null-safe)
            populateMdc(envelope);

            // Extract typed payload
            T payload = deserializer.extractPayload(envelope, payloadType);

            // Execute business logic
            processor.accept(payload);

        } catch (Exception e) {
            // Delegate error handling to caller
            errorHandler.accept(e, record);
        } finally {
            // CRITICAL: Clear MDC before returning to thread pool
            MDC.clear();
        }
    }

    /**
     * Populate MDC with trace fields from envelope.
     *
     * Null-safe: generates fallback IDs for missing trace.
     */
    private static void populateMdc(EventEnvelope envelope) {
        // Handle missing or null trace
        if (envelope.trace() == null) {
            MDC.put("requestId", "generated-" + UUID.randomUUID());
            MDC.put("correlationId", "generated-" + UUID.randomUUID());
        } else {
            MDC.put("requestId", Objects.requireNonNullElse(
                    envelope.trace().requestId(),
                    "generated-" + UUID.randomUUID()
            ));
            MDC.put("correlationId", Objects.requireNonNullElse(
                    envelope.trace().correlationId(),
                    "generated-" + UUID.randomUUID()
            ));
        }

        MDC.put("eventId", Objects.requireNonNullElse(
                envelope.eventId(),
                "unknown"
        ));
    }
}
```

**Why functional wrapper instead of abstract class:**
- No inheritance coupling (Java single inheritance limitation)
- Reusable across any consumer without class hierarchy
- Explicit error handling via callback
- Null-safe trace handling with generated fallbacks

```java
@Component
@Slf4j
public class InputReceivedConsumer {

    private final EventEnvelopeDeserializer deserializer;
    private final DecisionService decisionService;

    public InputReceivedConsumer(
            EventEnvelopeDeserializer deserializer,
            DecisionService decisionService) {
        this.deserializer = deserializer;
        this.decisionService = decisionService;
    }

    @KafkaListener(topics = "${triagemate.topics.input-received}")
    public void consume(ConsumerRecord<String, String> record) {
        MdcKafkaSupport.consumeWithMdc(
                record,
                InputReceivedEvent.class,
                deserializer,
                this::processEvent,
                this::handleError
        );
    }

    private void processEvent(InputReceivedEvent event) {
        log.info("event received eventType={}", event.type());
        decisionService.executeDecision(event);
        log.info("event processed");
    }

    private void handleError(Exception e, ConsumerRecord<String, String> record) {
        if (e instanceof EventDeserializationException) {
            log.error("deserialization failed offset={} partition={}",
                    record.offset(), record.partition(), e);
            // Send to DLQ or dead letter topic
        } else {
            log.error("processing failed offset={} partition={}",
                    record.offset(), record.partition(), e);
            // Retry or fail based on error type
        }
    }
}
```

**Key design decisions:**
- Business logic isolated in `processEvent()`
- Error handling isolated in `handleError()`
- MDC lifecycle managed by utility (single responsibility)
- Explicit separation of concerns

#### Goal

When writing to outbox, ensure trace fields from current MDC are embedded in the JSONB payload.

This enables MDC restoration during asynchronous publishing (task 11.2.c).

**Step 2: Create outbox event envelope builder**

**Create:** `com.triagemate.triage.outbox.OutboxEventBuilder`

```java
@Component
public class OutboxEventBuilder {

    private final ObjectMapper objectMapper;

    public OutboxEventBuilder(ObjectMapper objectMapper) {
        this.objectMapper = objectMapper;
    }

    /**
     * Build event envelope with trace from current MDC.
     *
     * This preserves correlation context for async publishing.
     */
    public <T> String buildEnvelope(String eventType, T payload) {
        // Extract trace from MDC
        String requestId = MDC.get("requestId");
        String correlationId = MDC.get("correlationId");

        // Build envelope
        EventEnvelope envelope = new EventEnvelope(
                UUID.randomUUID().toString(),  // New event ID
                eventType,
                ZonedDateTime.now(ZoneOffset.UTC),
                new EventEnvelope.Trace(requestId, correlationId),
                objectMapper.valueToTree(payload)
        );

        try {
            return objectMapper.writeValueAsString(envelope);
        } catch (JsonProcessingException e) {
            throw new OutboxSerializationException("Failed to serialize envelope", e);
        }
    }
}
```

**Why capture from MDC:**
- MDC populated at Kafka boundary (task 11.2.a)
- MDC available throughout transaction
- Trace automatically flows from input → outbox

5. Publisher can later extract trace from payload (task 11.2.c)

### 11.2.c — Restore MDC in Outbox Publisher

#### Goal

When `OutboxPublisher` publishes events asynchronously, restore trace fields from payload to MDC.

This enables correlated logs during Kafka publish operations.

**Critical caveat:** `kafkaTemplate.send()` is async and returns `CompletableFuture`. Logs from Kafka producer internals (acks, retries) run in Kafka's IO threads and won't have MDC. We can only correlate application-level logs in the scheduler thread.

---

#### Technical Implementation

**Step 1: Update publisher to restore MDC**

**Updated:** `com.triagemate.triage.outbox.OutboxPublisher`

```java
@Component
@Slf4j
public class OutboxPublisher {

    private final OutboxRepository outboxRepository;
    private final KafkaTemplate<String, String> kafkaTemplate;
    private final EventEnvelopeDeserializer deserializer;

    public OutboxPublisher(
            OutboxRepository outboxRepository,
            KafkaTemplate<String, String> kafkaTemplate,
            EventEnvelopeDeserializer deserializer) {
        this.outboxRepository = outboxRepository;
        this.kafkaTemplate = kafkaTemplate;
        this.deserializer = deserializer;
    }

    /**
     * Publish pending outbox messages.
     *
     * Single-threaded scheduled task.
     * If multi-instance deployment needed, use distributed lock or partitioning.
     */
    @Scheduled(fixedDelay = 1000)
    public void publishPending() {
        List<OutboxMessage> pending = outboxRepository.findPendingForPublish(100);

        for (OutboxMessage message : pending) {
            try {
                // Restore MDC from payload for correlated logging
                restoreMdcFromPayload(message);

                // Log before send (has MDC)
                log.info("publishing outbox message messageId={} eventType={}",
                        message.getId(), message.getEventType());

                // Send to Kafka (async - returns CompletableFuture)
                String topic = resolveTopicName(message.getEventType());
                CompletableFuture<SendResult<String, String>> future =
                        kafkaTemplate.send(topic, message.getPayload());

                // Block until send completes (keep MDC in scope)
                // Alternative: use callback with MDC copy
                SendResult<String, String> result = future.get(5, TimeUnit.SECONDS);

                // Mark as published
                outboxRepository.markPublished(message.getId(), ZonedDateTime.now(ZoneOffset.UTC));

                // Log after successful publish (has MDC)
                log.info("outbox message published partition={} offset={}",
                        result.getRecordMetadata().partition(),
                        result.getRecordMetadata().offset());

            } catch (TimeoutException e) {
                log.error("kafka send timeout messageId={}", message.getId(), e);
                // Retry on next poll
            } catch (Exception e) {
                log.error("publish failed messageId={}", message.getId(), e);
                // Retry logic or DLQ
            } finally {
                // Clear MDC before next message
                MDC.clear();
            }
        }
    }

    /**
     * Restore MDC from outbox payload.
     *
     * Non-critical: failure to restore doesn't prevent publish.
     */
    private void restoreMdcFromPayload(OutboxMessage message) {
        try {
            EventEnvelope envelope = deserializer.deserialize(message.getPayload());

            if (envelope.trace() != null) {
                MDC.put("requestId", Objects.requireNonNullElse(
                        envelope.trace().requestId(), "unknown"));
                MDC.put("correlationId", Objects.requireNonNullElse(
                        envelope.trace().correlationId(), "unknown"));
            }

            MDC.put("eventId", Objects.requireNonNullElse(
                    envelope.eventId(), "unknown"));

        } catch (Exception e) {
            log.warn("failed to restore MDC from payload messageId={}",
                    message.getId(), e);
            // Continue without MDC - better than failing publish
        }
    }

    private String resolveTopicName(String eventType) {
        return switch (eventType) {
            case "DECISION_EXECUTED" -> "triagemate.decision.executed.v1";
            default -> throw new IllegalArgumentException("Unknown event type: " + eventType);
        };
    }
}
```

**Key design decisions:**
- `future.get()` blocks to keep MDC in scope during publish
- Logs before/after send have MDC (application-level correlation)
- Kafka internal logs (producer IO threads) won't have MDC - this is expected
- MDC restoration failures don't prevent publish (logged as WARN)
- MDC cleared after each message in finally block

**Alternative - Async callback with MDC copy:**

```java
// Copy MDC before async callback
Map<String, String> mdcCopy = MDC.getCopyOfContextMap();

future.whenComplete((result, ex) -> {
        try {
        // Restore MDC in callback thread
        if (mdcCopy != null) {
        MDC.setContextMap(mdcCopy);
        }

                if (ex != null) {
        log.error("kafka send failed messageId={}", message.getId(), ex);
        } else {
        log.info("outbox message published partition={} offset={}",
                 result.getRecordMetadata().partition(),
                     result.getRecordMetadata().offset());
        outboxRepository.markPublished(message.getId(), ZonedDateTime.now(ZoneOffset.UTC));
        }
        } finally {
        MDC.clear();
    }
            });
```

**Trade-off:** Async callback increases complexity but avoids blocking scheduler thread.

**Current implementation:** Single-threaded `@Scheduled` task.

**Deployment scenarios:**

| Scenario | Lock needed? | Notes |

### 11.2.d — Propagate MDC to Async Tasks

#### Goal

If any part of the pipeline uses `@Async` or custom thread pools, propagate MDC to child threads.

Without this, async tasks lose correlation context.

---

#### Technical Implementation

**Step 1: Create MDC task decorator**

**Create:** `com.triagemate.triage.config.MdcTaskDecorator`

```java
import org.slf4j.MDC;
import org.springframework.core.task.TaskDecorator;

import java.util.Map;

/**
 * Decorator that propagates MDC to async tasks.
 *
 * Without this, @Async methods and ExecutorService tasks lose MDC context.
 */
public class MdcTaskDecorator implements TaskDecorator {

    @Override
    public Runnable decorate(Runnable runnable) {
        // Capture MDC from parent thread
        Map<String, String> contextMap = MDC.getCopyOfContextMap();

        return () -> {
            try {
                // Restore MDC in child thread
                if (contextMap != null) {
                    MDC.setContextMap(contextMap);
                }

                // Execute task with MDC
                runnable.run();

            } finally {
                // Clear MDC after task completes
                MDC.clear();
            }
        };
    }
}
```

```java
public class EventEnvelopeDeserializer {

    private final ObjectMapper objectMapper;

    public EventEnvelopeDeserializer(ObjectMapper objectMapper) {
        this.objectMapper = objectMapper;
    }

    public EventEnvelope deserialize(String json) {
        try {
            return objectMapper.readValue(json, EventEnvelope.class);
        } catch (JsonProcessingException e) {
            throw new EventDeserializationException("Failed to deserialize event envelope", e);
        }
    }

    public <T> T extractPayload(EventEnvelope envelope, Class<T> payloadType) {
        try {
            return objectMapper.treeToValue(envelope.payload(), payloadType);
        } catch (JsonProcessingException e) {
            throw new EventDeserializationException("Failed to extract payload", e);
        }
    }
}
```

```java
@Component
@Slf4j
public class InputReceivedConsumer {

    private final EventEnvelopeDeserializer deserializer;
    private final DecisionService decisionService;

    public InputReceivedConsumer(
            EventEnvelopeDeserializer deserializer,
            DecisionService decisionService) {
        this.deserializer = deserializer;
        this.decisionService = decisionService;
    }

    @KafkaListener(topics = "${triagemate.topics.input-received}")
    public void consume(ConsumerRecord<String, String> record) {
        try {
            // Step 1: Deserialize envelope
            EventEnvelope envelope = deserializer.deserialize(record.value());

            // Step 2: Populate MDC with trace fields
            populateMdc(envelope);

            // Step 3: Extract typed payload
            InputReceivedEvent event = deserializer.extractPayload(
                    envelope,
                    InputReceivedEvent.class
            );

            // Step 4: Execute business logic (MDC is now populated)
            log.info("event received eventType={}", envelope.eventType());
            decisionService.executeDecision(event);
            log.info("event processed");

        } catch (EventDeserializationException e) {
            log.error("deserialization failed", e);
            // DLQ handling or metrics increment
        } catch (Exception e) {
            log.error("processing failed", e);
            // Error handling
        } finally {
            // Step 5: CRITICAL - Clear MDC to prevent thread pool pollution
            clearMdc();
        }
    }

    private void populateMdc(EventEnvelope envelope) {
        MDC.put("requestId", envelope.trace().requestId());
        MDC.put("correlationId", envelope.trace().correlationId());
        MDC.put("eventId", envelope.eventId());

        log.debug("MDC populated requestId={} correlationId={} eventId={}",
                envelope.trace().requestId(),
                envelope.trace().correlationId(),
                envelope.eventId());
    }

    private void clearMdc() {
        MDC.clear();
        log.debug("MDC cleared");
    }
}
```

```java
public abstract class MdcKafkaConsumer<T> {

    private final EventEnvelopeDeserializer deserializer;

    protected MdcKafkaConsumer(EventEnvelopeDeserializer deserializer) {
        this.deserializer = deserializer;
    }

    /**
     * Template method enforcing MDC lifecycle.
     *
     * Subclasses implement processPayload() only.
     */
    protected void consumeWithMdc(ConsumerRecord<String, String> record, Class<T> payloadType) {
        try {
            EventEnvelope envelope = deserializer.deserialize(record.value());
            populateMdc(envelope);

            T payload = deserializer.extractPayload(envelope, payloadType);
            processPayload(payload);

        } finally {
            MDC.clear();
        }
    }

    private void populateMdc(EventEnvelope envelope) {
        MDC.put("requestId", envelope.trace().requestId());
        MDC.put("correlationId", envelope.trace().correlationId());
        MDC.put("eventId", envelope.eventId());
    }

    /**
     * Subclass implements business logic here.
     * MDC is guaranteed to be populated.
     */
    protected abstract void processPayload(T payload);
}
```

```java
@Component
@Slf4j
public class OutboxPublisher {

    private final OutboxRepository outboxRepository;
    private final KafkaTemplate<String, String> kafkaTemplate;
    private final EventEnvelopeDeserializer deserializer;

    @Scheduled(fixedDelay = 1000)
    public void publishPending() {
        List<OutboxMessage> pending = outboxRepository.findPendingForPublish(100);

        for (OutboxMessage message : pending) {
            try {
                // Step 1: Restore MDC from payload
                restoreMdcFromPayload(message);

                // Step 2: Publish to Kafka (logs now have MDC)
                log.info("publishing outbox event eventType={}", message.getEventType());
                publishToKafka(message);

                // Step 3: Mark as published
                outboxRepository.markPublished(message.getId());
                log.info("outbox event published");

            } catch (Exception e) {
                log.error("publish failed messageId={}", message.getId(), e);
                // Retry logic
            } finally {
                // Step 4: Clear MDC for next message
                MDC.clear();
            }
        }
    }

    private void restoreMdcFromPayload(OutboxMessage message) {
        try {
            // Deserialize JSONB payload to EventEnvelope
            EventEnvelope envelope = deserializer.deserialize(message.getPayload());

            // Populate MDC with trace fields
            MDC.put("requestId", envelope.trace().requestId());
            MDC.put("correlationId", envelope.trace().correlationId());
            MDC.put("eventId", envelope.eventId());

            log.debug("MDC restored from outbox payload");

        } catch (Exception e) {
            log.warn("failed to restore MDC from payload messageId={}",
                    message.getId(), e);
            // Continue without MDC - better than failing publish
        }
    }

    private void publishToKafka(OutboxMessage message) {
        String topic = resolveTopicName(message.getEventType());
        kafkaTemplate.send(topic, message.getPayload());
    }
}
```

**Key design decisions:**
- MDC restoration wrapped in try-catch (non-critical operation)
- Failure to restore MDC doesn't prevent publish
- MDC cleared after each message in finally block
- Debug log confirms MDC restoration

```java
                   payload::jsonb->>'eventId' as event_id
            FROM outbox_messages
            WHERE published_at IS NULL
            ORDER BY created_at ASC
            LIMIT :limit
            FOR UPDATE SKIP LOCKED
            """, nativeQuery = true)
    List<OutboxMessageWithTrace> findPendingWithTrace(@Param("limit") int limit);
}

public interface OutboxMessageWithTrace {
    Long getId();
    String getPayload();
    String getEventType();
    String getRequestId();
    String getCorrelationId();
    String getEventId();
}
```

**Usage:**

```java
private void restoreMdcFromProjection(OutboxMessageWithTrace message) {
    if (message.getRequestId() != null) {
        MDC.put("requestId", message.getRequestId());
    }
    if (message.getCorrelationId() != null) {
        MDC.put("correlationId", message.getCorrelationId());
    }
    if (message.getEventId() != null) {
        MDC.put("eventId", message.getEventId());
    }
}
```

**Trade-offs:**
- **Pro:** No deserialization overhead, direct JSONB extraction
- **Pro:** Single database round-trip
- **Con:** SQL becomes more complex
- **Con:** Requires projection interface

**Recommendation:** Start with Java deserialization (simpler), optimize with SQL extraction if performance profiling shows bottleneck.

```java
@SpringBootTest
@Testcontainers
@ActiveProfiles("docker")
class EndToEndCorrelationTest {

    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    @Test
    void correlation_shouldSpanEntirePipeline() {
        // Given
        String requestId = "req-e2e-" + UUID.randomUUID();
        String correlationId = "corr-e2e-" + UUID.randomUUID();
        String eventId = "evt-e2e-" + UUID.randomUUID();

        String envelope = createEnvelopeJson(requestId, correlationId, eventId);

        LogCapture logCapture = LogCapture.start();

        // When - Send event through pipeline
        kafkaTemplate.send("input-received", envelope);

        await().atMost(Duration.ofSeconds(10))
                .until(() -> logCapture.contains("outbox event published"));

        logCapture.stop();

        // Then - All logs have same correlation IDs
        List<JsonNode> logs = logCapture.getLogsAsJson();

        // Filter logs for this specific event
        List<JsonNode> eventLogs = logs.stream()
                .filter(log -> requestId.equals(log.get("requestId").asText()))
                .toList();

        assertThat(eventLogs).hasSizeGreaterThan(3); // At least: received, executed, published

        // All logs have consistent correlation
        eventLogs.forEach(log -> {
            assertThat(log.get("requestId").asText()).isEqualTo(requestId);
            assertThat(log.get("correlationId").asText()).isEqualTo(correlationId);
            assertThat(log.get("eventId").asText()).isEqualTo(eventId);
        });

        // Verify pipeline stages present
        assertThat(findLog(eventLogs, "event received")).isNotNull();
        assertThat(findLog(eventLogs, "decision executed")).isNotNull();
        assertThat(findLog(eventLogs, "publishing outbox event")).isNotNull();
        assertThat(findLog(eventLogs, "outbox event published")).isNotNull();
    }
```
